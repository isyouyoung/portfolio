# 캠퍼스 캡스톤 회고 — 빅데이터 + React

한국폴리텍대학 서울강서캠퍼스 빅데이터소프트웨어학과에서 진행 중인
캡스톤 프로젝트의 첫 한 달 회고입니다.

## 무엇을 만드는가

**강서구 골목상권 매출 대시보드**.
공공데이터포털의 상권 분석 데이터를 받아서, 동·업종별 매출과 점포 수
변화를 한눈에 보여주는 웹입니다.

## 역할 분담

- 데이터 수집/전처리 (Python·Pandas) — 팀원 A
- API 서버 (FastAPI) — 팀원 B
- **프론트엔드 (React)** — 내 담당
- 발표·문서화 — 팀원 D

## 프론트엔드에서 배운 것

### 1. 상태가 많을 때는 처음부터 Redux

처음엔 `useState`로 시작했는데, 필터 상태(지역·업종·기간)가
여러 컴포넌트에서 동시에 필요해지면서 prop drilling이 시작됐어요.
**3개 컴포넌트 이상에서 같은 상태**를 만지면 Redux로 옮기는 게 빠릅니다.

### 2. 서버 데이터는 React Query에 맡기기

`fetch`로 직접 로딩 상태를 관리하다가 모든 페이지에 똑같은 코드를
복붙하고 있다는 걸 깨달았습니다. React Query로 옮긴 뒤로:

- 같은 데이터 재호출 자동 캐싱
- `isLoading`, `isError` 같은 상태 무료
- 코드 양 절반

### 3. UI 라이브러리는 일찍 결정

Tailwind + 간단한 자체 컴포넌트로 갈지, shadcn/ui 같은 걸 쓸지를
1주차에 결정하지 못해서 시간이 많이 갔습니다. 다음엔 1주차 마지막에
**디자인 결정**을 한 번 픽스하고 가려고 합니다.
